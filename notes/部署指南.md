# 部署指南

## 架构说明

```
浏览器
  │  HTTP / WebSocket
  ▼
Nginx（80/443 端口）
  ├── /          → 静态文件（Vite 构建产物 dist/）
  ├── /api/      → 反向代理 → Node.js :3001
  └── /socket.io → 反向代理（含 WS 升级） → Node.js :3001
```

生产环境不使用 Vite Dev Server，前端构建成静态文件由 Nginx 托管，Node.js 只负责 API 和 Socket.IO。

---

## 一、服务器环境准备

```bash
# 安装 Node.js 20+
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 安装 Nginx
sudo apt install -y nginx

# 安装 PM2（进程守护）
sudo npm install -g pm2
```

---

## 二、上传代码

在本机项目根目录执行（替换 `user@your-server-ip`）：

```bash
# 排除 node_modules 和本地数据上传源码
rsync -avz --exclude='node_modules' --exclude='server/store/data/*.json' \
  ./ user@your-server-ip:/srv/fable5/
```

---

## 三、服务器上构建

```bash
cd /srv/fable5

# 安装依赖
npm install

# 构建前端静态文件 → 生成 dist/ 目录
npm run build
```

---

## 四、配置 Nginx

创建配置文件：

```bash
sudo nano /etc/nginx/sites-available/fable5
```

写入以下内容（把 `your-domain.com` 换成你的域名或服务器 IP）：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    root /srv/fable5/dist;
    index index.html;

    # SPA 路由兜底
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Socket.IO 反向代理（必须支持 WebSocket 升级）
    location /socket.io/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400s;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/fable5 /etc/nginx/sites-enabled/
sudo nginx -t          # 检查语法
sudo systemctl reload nginx
```

---

## 五、启动 Node.js 服务器

```bash
cd /srv/fable5

# 用 PM2 启动，进程名 fable5-server
pm2 start server/index.js --name fable5-server

# 设置开机自启
pm2 save
pm2 startup   # 按输出提示执行一条 sudo 命令
```

常用 PM2 命令：

```bash
pm2 status              # 查看进程状态
pm2 logs fable5-server  # 查看日志
pm2 restart fable5-server
pm2 stop fable5-server
```

---

## 六、开放防火墙端口

```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp   # 如果后续配 HTTPS
# Node.js 3001 端口不需要对外开放，Nginx 内部转发即可
```

---

## 七、配置 HTTPS（可选，推荐）

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

Certbot 会自动修改 Nginx 配置并设置证书自动续期。

---

## 八、后续更新部署

```bash
# 本机：上传新代码
rsync -avz --exclude='node_modules' --exclude='server/store/data/*.json' \
  ./ user@your-server-ip:/srv/fable5/

# 服务器：重新构建 + 重启
cd /srv/fable5
npm install
npm run build
pm2 restart fable5-server
```

---

## 九、存档数据备份

玩家数据存在 `server/store/data/` 目录下的 JSON 文件中，建议定期备份：

```bash
# 手动备份
cp -r /srv/fable5/server/store/data/ ~/backup-$(date +%Y%m%d)/

# 或加入 crontab 每天凌晨 3 点自动备份
0 3 * * * cp -r /srv/fable5/server/store/data/ ~/backups/fable5-$(date +\%Y\%m\%d)/
```

## 常见问题

| 现象 | 排查方向 |
|------|----------|
| 页面 502 Bad Gateway | `pm2 status` 确认 Node.js 进程在运行 |
| Socket.IO 连接失败 | Nginx `/socket.io/` 配置中 `Upgrade` 头是否缺失 |
| API 返回 404 | Nginx `/api/` location 是否正确，Node.js 日志有无报错 |
| 页面刷新 404 | Nginx `try_files` 兜底是否配置 |
| 存档丢失 | 检查 `server/store/data/` 目录权限，Node.js 进程用户是否有写权限 |
